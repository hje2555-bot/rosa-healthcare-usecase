# 05. Regulatory & Safety Considerations

## 1. 한국 의료법 가드레일

### 의료법 제27조 (무면허 의료행위 금지)
본 시스템이 절대 수행해서는 안 되는 행위:
- 진단(*"이건 X 질환입니다"*)
- 처방(*"Y 약을 드세요"*)
- 용량 조정(*"Y 약을 두 배로 드세요"*)
- 치료 행위 권유(*"수술을 받으셔야 합니다"*)

본 시스템이 허용되는 행위:
- 보건교육·건강 정보 제공
- 트리아지(우선순위 분류) — 단, 의료진에게 *전달*하는 행위로 한정
- 정서적 지지·CBT 자가관리 모듈 안내
- 의료진/리소스 라우팅

### 의료법 제22조, 제21조의2 (의무기록·전자의무기록)
- 본 시스템이 생성하는 환자 응대 로그는 *의무기록*이 아니라 *환자 자가관리 보조 로그*로 분류
- 의료진이 임상 결정에 활용한 부분만 EMR에 별도 기록
- 보관 기간: 환자 동의 기반 5년, 동의 철회 시 즉시 삭제

## 2. 식약처 SaMD (Software as a Medical Device) 분류

식약처 *의료기기로서의 소프트웨어 가이드라인(2022 개정)* 기준:

| Class | 정의 | 본 시스템 해당 여부 |
|-------|------|---------------------|
| Class 1 | 잠재적 위험성 거의 없음 | **해당** (현재 시점) |
| Class 2 | 잠재적 위험성 낮음 | 향후 응급 알림 정밀화 시 진입 가능 |
| Class 3 | 잠재적 위험성 중등도 | 진단 보조 진입 시 |
| Class 4 | 잠재적 위험성 높음 | 해당 없음 |

본 시스템 Phase 1~2는 Class 1 범위 내에서 운영하고, Class 2 진입 시점에 **식약처 사전 신고**를 통한 정식 SaMD 인증 절차 진행.

## 3. 개인정보보호법 + 가명정보 활용

### 수집 동의 모델
- 옵트인 명시 동의 (의료법 + 개인정보보호법 이중 동의)
- 동의 철회 즉시 데이터 삭제 (right to be forgotten)
- 가족 공유 옵션은 **별도 동의** 필요

### 가명·익명처리
- 자기반성 학습(Layer 5)에 사용되는 데이터는 *가명처리* 후 사용
- 모델 재학습 시에는 *익명처리* (개인정보보호법 제28조의2 활용)

### 데이터 저장 위치
- 1차 저장소: 한국 내 클라우드 (KISA 클라우드보안인증 CSAP 획득 사업자)
- 의료 마이데이터 연동 시: 마이데이터 사업자 인증 필요

## 4. 안전성 가드레일 (Self-reflection Layer 핵심)

### 4.1 응급 신호 자동 에스컬레이션
다음 키워드 또는 조합 감지 시 *즉시* 응급 프로토콜 작동:
- 자살·자해 직접 언급
- 흉통 + 호흡곤란 + 식은땀 (심근경색 의심)
- 의식 변화 + 두통 + 시야 이상 (뇌졸중 의심)
- 출혈 + 어지러움
- 항암 환자: 39도 이상 발열 (호중구감소성 발열)

응급 프로토콜 = 응급의료정보센터 1339 안내 + 가족 알림 + 의료진 즉시 호출.

### 4.2 모델 출력 게이트
모든 LLM 출력은 다음 3중 필터를 통과해야 환자에게 전달:
1. **약품명·용량 필터**: 약 이름이나 용량 숫자가 출력에 포함되면 차단
2. **진단명 필터**: ICD-10 코드 매칭 진단 표현 차단
3. **확정적 표현 필터**: *"확실히"*, *"틀림없이"*, *"바로"* 등 단정 표현 약화

### 4.3 인간 확인 게이트 (Human-in-the-loop)
다음 행동은 *반드시* 의료진 확인 후 실행:
- 의료진 알림 발송
- 외래 일정 변경 권유
- 응급실 방문 권유
- 가족에게 환자 상태 공유

자기반성 학습 결과(임계값 조정 등)도 *임상 책임자 검토 후 적용*. 완전 자동화 금지.

## 5. AI 거버넌스 표준 (참고)

- **WHO Ethics & Governance of AI for Health (2021)**: 6원칙 — 자율성 보호, 안전·복리 증진, 투명성·설명가능성, 책임소재, 형평성, 지속가능성
- **EU AI Act (2024)**: 의료 AI는 High-Risk 카테고리, CE 인증 + 위험관리시스템 필수
- **US FDA AI/ML SaMD Action Plan**: 사전결정 변경 통제 계획(PCCP) 도입

본 시스템은 Phase 0에서는 비규제 데모지만, Phase 1 이후 **WHO 6원칙을 설계 문서에 명시 통합**할 예정.

## 5-1. LG전자 ThinQ Privacy 정렬

LG ThinQ 생태계의 데이터·프라이버시 원칙과 본 시스템 설계의 정렬 항목.

| LG ThinQ 원칙 | 본 시스템 적용 |
|----------------|------------------|
| Data Minimization | 6채널 센서 중 *목적별로 필요한 채널만* 활성화 (예: 인지중재 모드에서는 LiDAR 비활성) |
| User Consent Control | 환자 + 보호자 별도 동의 모델, 가족 영상 공유 토글 분리 |
| On-device Processing First | 다약제 모니터링의 1차 신호 처리는 디바이스 내 수행, 클라우드 전송은 가공 결과만 |
| Auditability | 모든 알림·라우팅 결정에 rationale 로그 자동 첨부 (감사·소송 대비) |
| Data Sovereignty | 한국 내 클라우드 (KISA CSAP 인증 사업자), 환자 데이터 국외 이전 금지 |

LG가전 사업본부의 *ThinQ Care 생태계*와 본 시스템은 데이터 모델·동의 모델·감사 로그 구조를 공유하도록 설계된다.

## 5-2. Upstage AI Safety 정렬

Upstage Solar LLM이 의료 도메인에서 작동할 때의 안전 요건.

| Upstage Safety 항목 | 본 시스템 적용 |
|----------------------|------------------|
| Hallucination Mitigation | 의학 NER 후처리 게이트로 약품명·용량·진단명 출력 자동 차단 |
| Domain-specific Fine-tuning | 의료 도메인 instruction tuning (한국어 환자 자가보고 + 임상 지침서 기반) |
| Output Filtering | 진단 표현·확정적 표현·자살관념 응답 등 3중 필터 (LLM 출력 → 임상 게이트 → 사용자) |
| Evaluation Suite | 한국 의료법 27조 위반 케이스 1,000건 평가셋 통과 후 deploy |
| Rollback Capability | 문제 발생 시 24시간 내 이전 모델 버전 롤백 가능 구조 |

Upstage Solar의 의료 fine-tune 결과물은 본 프로젝트가 *first reference application*이 되며, 향후 LG 다른 가전 라인에 확산 가능한 안전 baseline을 제시한다.

## 6. 임상시험·IRB 절차 (Phase 2 진입 시)

- 첫 환자 파일럿: **연구목적 IRB 승인** 필수 (소속 연구병원 또는 대학병원)
- 동의서: 환자 + 보호자 동시 서명, AI 시스템 한계 명시
- 데이터 모니터링 위원회(DMC) 구성
- 사전등록(Clinical Research Information Service, CRIS)

## 7. 결론

ROSA-Healthcare의 안전성 설계는 *"AI가 무엇을 할 수 있는가"* 가 아니라 *"무엇을 하지 않을 것인가"* 의 정의가 더 중요하다. 의료법·개인정보보호법·SaMD 규정의 가드레일을 설계 단계부터 내장하고, 자기반성 학습 계층에서도 *완전 자동화된 자기 수정을 금지* 하는 것이 본 시스템의 안전 철학이다.
